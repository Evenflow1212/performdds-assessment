## Architecture: Two-Pass Workbook Generation

The system uses a **two-pass** approach because ExcelJS corrupts template styles:

### Pass 1 — Value Injection (ExcelJS + JSZip)
1. `sv(worksheet, address, value)` collects values into `_cellCollector[sheetName][address]`
2. `injectValuesIntoTemplate()` (line 348) opens template XLSX as JSZip
3. For each sheet, finds existing `<c>` elements by cell reference and replaces values
4. Preserves original `s=""` style attribute from template
5. Strings → `<is><t>text</t></is>` (inlineStr), Numbers → `<v>123</v>`
6. New cells (not in template) get inserted with appropriate styles

### Pass 2 — Style Restoration & Formatting Fixes (JSZip XML)
1. Restores original `styles.xml` from template (ExcelJS reduces cellXfs from 752+ to ~400)
2. Normalizes fonts to Candara
3. Adds custom styles: strikethrough, SWOT, Batting Average box, Collections by Payor
4. Per-sheet fixes: column widths, row heights, IFERROR wrappers, hidden rows
5. Sheet-specific logic (e.g., clearing placeholder names in Employee Costs)

### The sv() Function (Line 27)
```javascript
function sv(sheetName, addr, val) {
  const name = typeof sheetName === 'string' ? sheetName : (sheetName?.name || sheetName);
  if (!_cellCollector[name]) _cellCollector[name] = {};
  if (val !== null && val !== undefined) {
    _cellCollector[name][addr] = val;
  }
}
```
ALL cell writes go through sv(). It does NOT write to ExcelJS — it populates `_cellCollector` which Pass 1 reads during XML injection.

## Hub Wizard Steps & Data Flow

### Step 1: Production (PDF upload, REQUIRED)
- Extracted via `extractProductionFromPdf()` → pipe-delimited text
- Format: `CODE|Description|Quantity|Total`
- Parsed by `parseProduction()` → `{ codes[], months, years[] }`

### Step 2: Collections (PDF upload, REQUIRED)
- Extracted via `extractCollectionsFromPdf()` → pipe-delimited text
- Format: `CHARGES|amount`, `PAYMENTS|amount`, `DATES|from - to`
- Parsed by `parseCollections()` → `{ charges, payments, months }`

### Step 3: P&L (PDF upload, optional)
- Extracted via `extractPLFromPdf()` → pipe-delimited text with sections
- Format: `SECTION|Income`, `ItemName|Amount`, `TOTAL_INCOME|amount`
- Parsed by `parsePL()` → `{ items[], totalIncome, totalExpense, netIncome }`
- Also renders to base64 JPEG for embedding

### Step 4: Patient AR (PDF upload, optional)
- Extracted to `{ current, d3160, d6190, d90plus, insr, total }`

### Step 5: Insurance AR (PDF upload, optional)
- Extracted to `{ current, d3160, d6190, d90plus, total }`

### Step 6: Employee Costs (form entry, optional)
- `collectEmployeeCosts()` → employeeCosts object
- Staff positions: om, f1-f5, b1-b5 (11 positions)
- Hygiene positions: h1-h5 (5 RDH slots)
- Each: `{ key, label, name, rate, hours }`
- Also: staffBenefits, staffEmpCostPct, hygBenefits, hygEmpCostPct, benefits{}

### Step 7: Hygiene Schedule (form entry, optional)
- `collectHygieneSchedule()` → hygieneData object
- rdhPerDay: [mon, tue, wed, thu] RDH count per day
- ptsPerHygDay: expected patients per hygienist
- recentPast: 3 weeks × 12 values (6 days × appt/seen)
- next7Days: 1 week × 12 values
- nearFuture: 3 weeks × 12 values
- futureFuture: 2 weeks × 12 values

### Step 8: Generate
- POST to `/.netlify/functions/generate-workbook`
- All data sent as JSON (no FormData/multipart)
- Response: `{ success, xlsxB64, summary }`
- Client downloads as `{practiceName}_Assessment_Workbook.xlsx`

### Zero-Filling (Line 2119)
Runs BEFORE derivation. Fills rows 10-12, 19-20, 25-27, 34-35 with 0 via sv() ONLY if cell not already in _cellCollector.

**CRITICAL BUG PATTERN:** Zero-filling populates _cellCollector with 0 values. Any subsequent code that checks `if (!hsCollector[cell])` will find the cell EXISTS (with value 0) and SKIP writing. This was the v45 bug — derivation code must write UNCONDITIONALLY (no hsCollector guards).

## Bug History & Lessons Learned

### v42: Empty Array Truthy Bug
**Problem:** `![]` is `false` in JavaScript — empty arrays are truthy. Code checked `if (!employeeCosts.staff)` which passed even when staff was `[]`.
**Fix:** Check `Array.isArray(x) && x.length > 0` instead of just `!!x`.
**Lesson:** ALWAYS check `.length` on arrays. `![]` → false, `![1]` → false. Both are truthy.

### v43: Pass 2 Clearing Overwrites Pass 1 Data
**Problem:** Pass 2 sheet6 unconditionally cleared D27/D28 (hygienist name cells), erasing names written by sv() in Pass 1.
**Fix:** Check `_cellCollector['Employee Costs']['D27']` before clearing. Only clear if Pass 1 didn't write there.
**Lesson:** Pass 2 XML manipulation can destroy Pass 1 injected values. Always check _cellCollector before clearing.

### v43: Hygienist Rate Too Low
**Problem:** Default hygienist rate was $40/hr, Dave wanted $50/hr.
**Fix:** Changed `avgHygRate` from 40 to 50.

### v44: P&L Wages Regex Anchor Bug
**Problem:** Regex `^wages?\b` used `^` anchor which fails when parsed PDF labels have leading whitespace.
**Fix:** Changed to `\bwages?\b` (word boundary). Also added `.trim()` to label before testing.
**Lesson:** PDF-parsed text often has leading whitespace. Never use `^` anchors on parsed PDF data. Use `\b` word boundaries.

### v44: Hygiene Truthy Bugs (Same Class as v42)
**Problem:** `hygieneData.recentPast` was `[]` (truthy), `hygieneData.rdhPerDay` was `[1,1,1,1,0,0]` (truthy with default 1s), `daysScheduled` form default was 8 (overriding calculated 252).
**Fix:**
```javascript
const hasRealRDH = hygieneData && Array.isArray(hygieneData.rdhPerDay) && hygieneData.rdhPerDay.some(v => v > 1);
const hasRealRecentPast = hygieneData && Array.isArray(hygieneData.recentPast) && hygieneData.recentPast.length > 0 && hygieneData.recentPast.some(w => w.data && w.data.some(v => v > 0));
const hasRealDaysScheduled = hygieneData && hygieneData.daysScheduled && hygieneData.daysScheduled > 20;
```
**Lesson:** Form defaults (empty arrays, default values like 1 or 8) must be distinguished from real user input. Check for meaningful data, not just existence.

### v45: Zero-Fill Prevents Derivation Overwrite
**Problem:** Zero-filling code (line 2119) runs FIRST and writes 0 to all grid cells. Derivation code (line 2258) checked `if (!hsCollector[cell])` which found the 0 already there and skipped.
**Fix:** Removed `if (!hsCollector[...])` guards. Derivation writes UNCONDITIONALLY with sv(), overwriting the zeros.
**Lesson:** When using a "fill defaults then override" pattern, the override code MUST NOT check if the cell exists — it must always overwrite. The _cellCollector will have the default value, making existence checks useless.

## Key Gotchas Summary

1. **JavaScript truthy:** `![]` is `false`. Empty arrays are truthy. Always check `.length`.
2. **PDF whitespace:** Parsed PDF text has leading whitespace. Use `\b` not `^` in regex. Use `.trim()`.
3. **Zero-fill then override:** Don't guard overrides with existence checks on _cellCollector.
4. **Pass 2 clears Pass 1:** Check _cellCollector before clearing template placeholders in Pass 2.
5. **Form defaults vs real data:** Default values (1, 8, empty arrays) look like real data. Check for meaningful values.
6. **openpyxl reads stale cache:** Formula cells show 0 in openpyxl. Open in Excel to see real values.
7. **zsh history expansion:** Use single quotes for git commit messages containing `!`.
8. **Sandbox can't push:** Copy git commands to clipboard for Dave to paste in Terminal.
9. **Netlify deploy timing:** ~21 seconds. Verify deploy is published before regenerating workbook.
